# Behavioral Lie Detection System

A full-stack AI interrogation platform that estimates deception risk from behavioral timing, semantic consistency, and contradiction patterns during scenario-driven questioning.

## Disclaimer

**This system estimates behavioral and linguistic patterns. It does not determine absolute truth.**

## System Architecture

- **Frontend:** React + Vite + Tailwind CSS + Framer Motion
- **Backend:** FastAPI REST API
- **ML Layer:** scikit-learn model training and inference bundle

## Core Workflow

1. User selects scenario:
   - Theft Investigation
   - Academic Dishonesty
   - Workplace Incident
2. UI starts with base scenario questions and then generates adaptive counter-questions.
3. System captures:
   - Response text
   - Response time
   - Typing duration
4. Backend extracts context memory (`location`, `time`, `people`, `actions`) from each response.
5. Contradiction engine flags timeline/context conflicts and maintains contradiction score.
6. Backend computes enhanced features:
  - `avg_response_time`
  - `variance_response_time`
  - `response_length_variation`
  - `behavioral_consistency`
  - `semantic_consistency`
  - `contradiction_score`
  - `hesitation_score`
7. Final decision engine combines:
  - `40%` behavioral component
  - `40%` semantic consistency component
  - `20%` contradiction component
8. ML model and weighted decision together produce probability and Low/Medium/High risk.
6. Dashboard renders metrics, probability gauge, final classification, and explanation factors.

## Machine Learning Approach

### Features

The model uses the following engineered features:

- `avg_response_time`: mean response latency
- `variance_response_time`: variation in response latency
- `response_length_variation`: variability in response lengths
- `behavioral_consistency`: lexical consistency across answers
- `semantic_consistency`: semantic overlap consistency across responses
- `contradiction_score`: normalized contradiction pressure
- `hesitation_score`: filler-word and pause-based hesitation index

### Models Trained

- Logistic Regression (with StandardScaler)
- Random Forest Classifier

Training compares both models on ROC-AUC, F1, and accuracy, then saves the best model bundle to:

- `backend/models/lie_model.pkl`

The bundle also stores feature statistics and feature importance metadata for explanation generation.

## API

### `POST /predict`

Request body:

```json
{
  "responses": ["...", "..."],
  "times": [4.2, 5.0],
  "typing_durations": [2.0, 2.4]
}
```

Response:

```json
{
  "probability": 0.78,
  "risk": "High",
  "contradictions": 2,
  "metrics": {
    "average_response_time": 5.3,
    "response_time_variance": 1.2,
    "response_length_variability": 0.34,
    "behavioral_consistency": 0.59,
    "semantic_consistency": 0.48,
    "contradiction_score": 0.62,
    "hesitation_indicators": 0.48
  },
  "explanation": [
    "Contradictory statements detected across timeline and context memory."
  ],
  "model_used": "random_forest",
  "disclaimer": "This system estimates behavioral and linguistic patterns. It does not determine absolute truth."
}
```

### `POST /followup`

Generates adaptive follow-up questions based on contradiction, vagueness, and missing details.

Request body:

```json
{
  "scenario": "theft",
  "history": [
    { "question": "...", "response": "..." }
  ]
}
```

Response includes:

- `followup_question`
- `analysis.semantic_consistency`
- `analysis.contradiction_count`
- `analysis.memory`

## Local Setup

### 1. Train Model (ML pipeline)

```bash
cd ml
pip install -r requirements.txt
python generate_data.py
python train_model.py
```

### 2. Start Backend

```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app:app --reload --port 8000
```

### 3. Start Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend runs on `http://localhost:5173` and expects backend on `http://127.0.0.1:8000`.

Optional env variable for frontend API base:

- `VITE_API_BASE_URL`

## UX Highlights

- Dark AI dashboard visual style
- Chat-style interrogation layout with visual hierarchy
- Adaptive question/answer/follow-up thread
- Live typing-duration tracking
- Contradiction highlighting in red responses
- Live AI analysis panel (semantic score, contradiction count, context memory)
- Demo mode (truthful vs deceptive) for competition walkthroughs
- Animated loading and result reveal
- Restart and full reset support

## Limitations

- Uses synthetic training data; not validated on real-world forensic datasets
- Behavioral cues are probabilistic and context-sensitive
- No audio/video biometric features (voice stress, gaze, facial microexpressions)
- Explanation is feature-attribution style, not causal truth detection
